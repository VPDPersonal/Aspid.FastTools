# EnumValues&lt;TValue&gt; — пошаговый туториал

`EnumValues<TValue>` — сериализуемый словарь, который сопоставляет члены любого enum значениям любого
сериализуемого типа и настраивается в Inspector. Каждый урок соответствует одной `STEP`-секции компонента
`EnumValuesTutorial`.

Объявите поле только с типом значения — тип enum выбирается в Inspector:

```csharp
// Какой enum? Решается в Inspector.
[SerializeField] private EnumValues<float> _multipliers;
```

Когда enum *известен* на этапе компиляции, используйте типизированный близнец `EnumValues<TEnum, TValue>`
(урок 5).

## Открыть туториал

1. Откройте окно Welcome (**Tools → Aspid 🐍 → FastTools → Welcome**) и импортируйте пример **EnumValues**.
2. Откройте **`Scenes/EnumValuesTutorial.unity`** и выберите GameObject **EnumValues Tutorial**.

---

## Урок 1 — Первое сопоставление

**Поле:** `EnumValues<float> _step1Multipliers` — пустое.

1. Кликните поле типа в шапке и выберите **`DamageType`** в пикере типов.
2. Нажмите **+** на списке. Появится строка: **дропдаун ключа** плюс **float-значение**.
3. Добавьте несколько строк, например `Physical = 1.0`, `Fire = 1.5`.

- Пока тип enum не выбран, поле показывает предупреждение, а каждый поиск возвращает значение по умолчанию.
- Inspector позволяет дублировать ключи, но поиск всегда выигрывает **первая** строка с данным ключом.
- Сторона значения — обычное сериализуемое поле: подойдёт любой сериализуемый тип (`Color`, `Sprite`,
  свой `[Serializable]`-класс…).

## Урок 2 — Populate Missing Enum Members и значение по умолчанию

**Поле:** `EnumValues<Color> _step2Colors` — тип enum уже выбран (`DamageType`), список пуст.

1. **Кликните правой кнопкой** по полю и выберите **Populate Missing Enum Members** — появится по строке на
   каждый член, заполненной **Default Value** (белым).
2. Перекрасьте пару строк: `Fire` — оранжевый, `Ice` — голубой.
3. Кликните правой кнопкой ещё раз — пункт меню неактивен: недостающих членов нет.

- **Default Value** (последнее поле) — одновременно затравка для добавляемых строк и fallback, который
  `GetValue` возвращает, когда ни одна запись не совпала. Строка на каждый член **не обязательна** —
  сопоставьте исключения, остальное покроет default.
- Populate только дописывает недостающие строки; существующие не трогает.

## Урок 3 — `[Flags]`-ключи и правила поиска

**Поле:** `EnumValues<float> _step3SpeedByStatus` — предзаполнено, ключ — `[Flags]` enum `StatusEffect`.

Каждый ключ отображается **мультиселект-дропдауном флагов**; комбинация вроде `Burning | Slowed` — обычная
самостоятельная запись. Предзаполненный список:

| # | Ключ | Значение |
|---|---|---|
| 0 | `Burning` | `0.9` |
| 1 | `Frozen` | `0.2` |
| 2 | `Slowed` | `0.5` |
| 3 | `Burning \| Slowed` | `0.4` |

`GetValue` разрешает `[Flags]`-поиск в три шага:

1. **Точное совпадение выигрывает первым, независимо от порядка** — поиск ровно `Burning | Slowed` возвращает
   `0.4`, хотя эта запись стоит последней.
2. **Нет точного совпадения → выигрывает первая содержащаяся запись** — у `Burning | Frozen | Slowed` точной
   записи нет, поэтому побеждает первая строка, все флаги которой содержатся в значении: `Burning`, `0.9`.
3. **Ничего не совпало → Default Value** — у `Stunned` записи нет: `1.0`. `None` (ноль) совпадает только с
   записью `None`.

## Урок 4 — Чтение значений из кода

**Поля:** `DamageType _step4DamageType`, `StatusEffect _step4Effects` — ключи поиска.

```csharp
float multiplier = _step1Multipliers.GetValue(_step4DamageType);
float speed      = _step3SpeedByStatus.GetValue(_step4Effects);
```

Кликните правой кнопкой по заголовку компонента → **Log Tutorial Lookups** (работает в Edit Mode). Меняйте
`_step4Effects`, чтобы проверить три правила из урока 3: `Burning | Frozen | Slowed` → `0.90`, `Stunned`
или `None` → `1.00`.

- `GetValue` принимает любое значение настроенного enum — регистрировать ключи в коде не нужно.
- `foreach` по `EnumValues` перебирает настроенные пары `(ключ, значение)` в порядке списка; значение по
  умолчанию в итерацию **не входит**.

## Урок 5 — Типизированный вариант: `EnumValues<TEnum, TValue>`

**Поле:** `EnumValues<DamageType, float> _step5TypedMultipliers` — предзаполнено.

Когда тип enum известен на этапе компиляции, объявите его прямо в поле вместо выбора в Inspector:

```csharp
// Enum зафиксирован на этапе компиляции.
[SerializeField] private EnumValues<DamageType, float> _multipliers;

// GetValue принимает DamageType, а не упакованный Enum.
float m = _multipliers.GetValue(DamageType.Fire);
```

- Строка выбора типа в шапке **заблокирована** — enum зафиксирован объявлением поля, а `GetValue` безопасен
  на этапе компиляции. Кнопка открытия скрипта рядом с ней остаётся активной.
- Всё из уроков 1–4 переносится: *Populate Missing Enum Members*, fallback на Default Value, правила
  `[Flags]`-поиска, `foreach` по записям.
- Сериализованный layout совпадает с `EnumValues<TValue>`: переключение поля между двумя вариантами (с тем же
  настроенным enum) сохраняет записи.
- **Правило выбора:** enum известен заранее → `EnumValues<TEnum, TValue>`; enum выбирается на стороне данных
  в Inspector → `EnumValues<TValue>`.

## Когда enum меняется

Ключи хранятся **строками имён**, поэтому:

- **Переименованный / удалённый член** — сохранённый ключ больше не парсится. Inspector мигрирует строку на
  первый член enum; в рантайме неразрешённый ключ логирует ошибку и просто никогда не совпадает (без краша).
- **Добавленный член** — у нового члена просто нет строки (→ значение по умолчанию), пока вы её не добавите,
  например через *Populate Missing Enum Members*.
- **Смена типа enum** на заполненном списке сохраняет строки и их значения и пере-парсит каждый ключ против
  нового типа — ключ, чьё имя существует в обоих типах, выживает; остальные мигрируют (и сохраняются) на
  первый член нового типа, так что при обратном переключении исходные имена ключей **не** вернутся.

> Если переименование должно сохранить данные — переименуйте сериализованные строки в файле ассета/сцены (или
> заново выберите ключ в Inspector): миграция ключа никогда не сбрасывает сторону значения.
